## 1. 한 문장 정체성

**36.5는 실패를 없애는 앱이 아니라, 실패 후 복귀율을 높이는 앱이다.**

- 시그니처: **"사람은 감싸되, 핑계는 감싸지 않는다"** — 사람에게는 따뜻함, 행동에는 사실 직면.
- 출시 포지셔닝: "작은 성공의 반복으로 자기신뢰를 쌓는 회복 루틴 앱".
- 모티브 문구(첫날 노출): "습관은 의지가 아니라 리듬으로 만들어요. 의지는 소모되지만, 리듬은 다시 돌아오게 해요."

## 2. 온도(코칭 머리디안) 컨셉

이 앱의 핵심 개인화 장치. **설문 → 온도 → 앱 전체가 그 온도로 반응**하는 구조.

1. 온보딩 33문항(11카테고리) + MBTI → `traitProfile` 산출
2. `coachingMeridian` = **36.1°~36.9°** 사이의 온도 1개
3. 온도 → 밴드 3종: `fact`(≤36.4, 사실 중심) / `balanced`(36.5) / `warm`(≥36.6, 온기 중심)
4. 홈 응원 문구, 인터럽터, 일기 프롬프트, 결과지 등 **모든 카피가 밴드에 반응**

밴드는 **말투(register)만 바꾸고, 시키는 행동은 동일**해야 한다.

## 3. 결과 페이지 컨셉 — "조각 합성" 자체가 정체성이다

결과 페이지는 고정 템플릿이 아니라 **동적 초상(dynamic portrait)** 이다:

- `portraitPhraseLibrary`의 카테고리별 문장 조각을 사용자 점수 rank에 따라 **합성**해 identityLine·감정 트리거·약속 문장을 만든다.
- 심화 해설(`deepAnalysis`): 답변 근거 해석 카드, 카테고리 교차 해석, 검증된 개입(EBI), 성장 청사진.
- 재측정 시 "지난번과 비교" 서술.

**중요**: 이 조각-합성 문체(다소 시적이고 단정적인 표현, 제품 고유 어휘)는 **버그가 아니라 의도된 컨셉**이다. "더 자연스럽게" 다듬는 것은 컨셉 훼손이다.

### 제품 고유 어휘 (교체 금지)
온도(36.1~36.9°) · 정화 일기 · 고요 포인트(GYO) · 복귀 · 흔들림 · 리듬 등 — 일반어로 바꾸지 않는다.

## 4. 카피 톤 규칙 (하드 제약)

- 문장 골격: **사람 수용 → 행동 교정 → 복귀 제안 → 동기**
- 톤 비율: 따뜻함 70 / 단호함 30
- 금지: 비난, 낙인, 훈계, 공포 조장, 비교 유도
- 필수: 공감, 사실 직면, 즉시 실행 가능한 최소 행동, 복귀 강조

## 5. 수정 허용 경계 — "알 수 없는 단어" 판별 기준

결과 페이지 카피 수정 요청이 있을 때, **아래 4가지만** 버그로 취급해 고친다:

| 허용 (버그) | 예 |
|---|---|
| 명백한 오타 | 맞춤법·타이핑 실수 |
| 깨진 문장 | 조각 결합 결과 조사·어미가 문법적으로 성립하지 않는 경우 |
| 내부 값 노출 | 영문 키, 카테고리 식별자, `undefined`/`null`, 플레이스홀더가 화면에 그대로 보이는 경우 |
| 기계적 중복 | 같은 온도·같은 문장이 한 화면에 두 번 렌더되는 등 |

| 금지 (컨셉) | 예 |
|---|---|
| 문체 다듬기 | "더 자연스러운 표현으로" 리라이팅 |
| 구조 변경 | 조각-합성 방식을 고정 문장으로 교체 |
| 어휘 일반화 | 제품 고유 어휘를 일반어로 교체 |
| 밴드 통일 | fact/balanced/warm의 말투 차이를 없애는 것 |

### 수정 절차
1. 사용자가 본 **실제 화면 문자열**(런타임 증거)을 먼저 확보한다 — 코드만 보고 추정 수정 금지.
2. 위 표로 허용/금지 분류.
3. 허용 범주만 **최소 범위**로 수정하고, 반영 전 사용자 컨펌.
